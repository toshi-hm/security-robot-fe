# Report ディレクトリについて

このディレクトリは、プロジェクトの進捗管理と作業記録を保存するためのものです。

## ファイル構成

### `progress.md`
- **目的**: プロジェクト全体の進捗状況を追跡
- **内容**:
  - 実装済み機能
  - 実装中の機能
  - TODO（未実装）リスト
  - 技術的課題・検討事項
- **更新頻度**: 各セッションで更新

### `sessions/`
- **目的**: 各作業セッションの詳細な記録を保存
- **ファイル命名規則**: `YYYY-MM-DD_session_XXX.md`
- **内容**:
  - セッション開始時刻
  - 実施内容
  - 作成・変更したファイル
  - 解決した課題
  - 残っている課題
  - 次のセッションで実施すべきこと
- **更新頻度**: 各セッション終了時に作成

## 作業フロー

1. **セッション開始時**
   - `progress.md` を読んで現在の進捗を確認
   - `sessions/` の最新セッション記録を読んで前回の作業内容を把握

2. **作業中**
   - 実装を進める
   - 必要に応じて `progress.md` を更新

3. **セッション終了時**
   - `progress.md` を最新の状態に更新
   - 新しいセッション記録ファイルを作成
   - 次回の作業内容をメモ

## 注意事項

- 進捗ファイルは編集可能です（常に最新の状態を反映させてください）
- セッション記録は基本的に追記専用です（後から参照するため）
- すべてのファイルはMarkdown形式で記述してください
